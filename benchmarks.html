How to read this

Each bar is one experiment: the same fixed set of prompts, run many times on the network, compared against a baseline run of the same prompts. The bar's length is how far the responses moved from the baseline's — 1 on this scale is the noise floor, the amount identical setups naturally wobble, so anything left of that line is "no real change" and anything past it is genuine behavioral drift, measured in multiples of that natural wobble. The label on each bar says what was deliberately different (derived from the experiments' signed configurations — computed, not written). Dots mark individual prompt groups: "peak" is the most-moved one, "breadth" is how many moved.

Why you can trust it

Every number is published by the researcher who ran the experiment, cryptographically signed, and anchored in a public transparency log — this page re-checks each signature in your browser, and the links under every bar go to the public log records. Nothing here is hand-entered or editable: results are admitted to this page automatically, by verification, not by an editor. To re-derive any number yourself from the underlying evidence: pip install auspexai-tenant · auspexai-tenant benchmark verify-entry — the raw registry is here.
